# コンテキスト生成（コンテキストエンジニアリング用）

## 概要
コンテキストエンジニアリング用のコンテキスト生成を行う。

## 必要事項の洗い出し
1. ユーザの指示及び目的を理解する
2. 各エージェントとやり取りを行い、コンテキストファイルを記載
3. コンテキストレビュー
4. 2～3を最大3回繰り返す

## エージェント
各エージェントを使用し、適切に処理を行う。
各エージェントとやり取りをすることで、適切なコードやレビューを取得し、コンテキストファイルに記載していく。
- agent document-explorer: 適切なファイルの取得及び要約
- agent code-explorer: コードのread、要約、レビュー

## コンテキストレビュー
agent context-review を起動して、コンテキストのレビューを行うこと。
最大3回まで修正を行う。

## 注意事項
- 最終決定権はユーザにある。判断に迷う場合はユーザに尋ねること
- **NEVER**: あなたができるのは各エージェントの立ち上げと指示のみ。勝手にコンテキストファイル以外のファイルを読み書きしたり、コマンドを叩くのは許可しない

## 出力
作成したコンテキストを、`.context/<日時:20251012.160422>.md`の形式で保存。

###  出力例
```markdown
# ログイン処理実装

## 概要
ログイン処理の実装を行う。

## 用語説明
- **SHA256**: ...
- **ハッシュ化**: ...

## 実装方針
- TDDによる実装

## 実装を行うファイル
- src/main/Login.py（新規）
- src/test/LoginTest.py（新規）
- src/main/Pattern.py（追加修正）
- src/test/PatternTest.py（追加修正）

## TDD REDフェーズのテストケース一覧
### LoginTest.py
1. ログインが成功すること
    - **given**: username=`username`, password=`password`
    - **when**: Login.login(username, password)
    - **then**: ログイン成功、200が返ってくる
2. ユーザ名バリデーションエラー
    - **given**: username=`username`, password=`p`
    - **when**: Login.login(username, password)
    - **then**: バリデーションエラー、403

## 使用するエージェント
1. tdd-red-agent: TDD REDフェーズで使用するエージェント
2. tdd-green-agent: TDD GREENフェーズで使用するエージェント

## 実装コードコーディング規約
- 1行は120文字以内に収めること
- 関数名はスネークケース、変数名はキャメルケース

## テストコードコーディング規約
- 文字数に関係あるものは`"a".repeat(150)`のように表記すること
- given-when-thenで実装すること

## 終了条件
- [ ] ログインの実装が完了していること
- [ ] コンパイルが正常に終了すること

## 参考ドキュメント
- `docs/requirements.md`
- `docs/design.md`
```

### 表示
`/action-context .context/<日時:20251012.160422>.md`を実行してください、と表示。
